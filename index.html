Dev Log

2026-03-14
todo-app ux bugfix

The dashboard now lets you hide any of the four quadrants you don't need. A small eye icon in each quadrant header toggles visibility, and the grid reflows so the remaining quadrants fill the space cleanly.

This started as a bug report — the health tracker was getting in the way for people who only wanted task management. Turned out the fix was simpler than expected: one hidden attribute on the quadrant wrapper and a grid-template-columns recalculation in CSS.

State is persisted in localStorage so your layout survives a page refresh.

2026-02-28
todo-app mobile css

The header on small screens was cramped — the nav labels were truncating and the dark mode toggle was falling off the edge. This pass shortened the labels, made the header always visible (no more hiding on scroll), and tightened the responsive breakpoint logic.

The main change was switching from a fixed pixel breakpoint to clamp() for font sizes and using gap instead of fixed margins between header items. Fewer magic numbers, more predictable behavior across device widths.

2026-02-10
todo-app testing playwright

Set up Playwright for the todo app with both desktop and mobile test projects. The suite covers the core flows: onboarding, adding and completing tasks, dark mode toggle, and the workout logger modal.

Since the app is zero-build and lives in a single HTML file, the test setup is minimal — python3 -m http.server as the web server, no compile step. Playwright spins it up automatically before each run.

npx playwright test --project=mobile

Screenshots are saved to test-results/ on failure, which has already caught two layout regressions that weren't visible on desktop.
